Future Session States

Exploring how the meso-grid and workout header should behave when a user navigates to a session that isn't the "current" one. Covers visual indicators, interaction models, and locked/preview states.

Outer ring on viewed cell

Recommendations

Grid indicator
Ring highlight (A). Subtle pulsing slate ring around the viewed cell. Doesn't compete with the blue "current" pulse. Works on all cell types.
Header treatment
"Preview" badge replaces the RiR badge. Progress bar goes dim/empty. Subtitle shows week + RiR info instead.
Interaction model
View-only for v1. Future sessions are read-only. All inputs disabled. "Go to Current Workout" CTA at bottom. Prevents progression data issues.
No-data state
Show dashes (—) for weights that haven't been computed yet. Keep target reps visible since those come from the split template and are always available.
Navigation flow
Tap future cell in grid → opens preview. Tap "Go to Current Workout" → navigates to current. Tap a done cell → opens the existing WorkoutDetail view (no change).
Implementation note
The Workout page already accepts ?week=&session= params. Add a isFutureSession check by comparing against get_current_position(). If future → disable inputs, swap header badges, show banner.
